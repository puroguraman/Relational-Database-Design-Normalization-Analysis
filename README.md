📝 Project Overview:This repository contains a comprehensive analysis and design of a Relational Database System. Developed for the "Relational Databases" course at the University of Piraeus, this project focuses on the transition from real-world requirements to a fully normalized logical schema. It demonstrates deep knowledge of data consistency, relationship modeling, and structural integrity.

🚀 Key Learning Outcomes & FeaturesEntity-Relationship Modeling (ERD): 
Mapping complex business rules into visual entities and relationships.
Database Normalization: Applying 1NF, 2NF, and 3NF rules to eliminate data redundancy and update anomalies.
Relational Algebra: Understanding the mathematical foundation behind data retrieval and manipulation.
Schema Refinement: Transforming conceptual models into optimized SQL tables with primary and foreign keys.

🛠️ Methodology & ToolsModeling Language: UML / Crow's Foot Notation for ER Diagrams.
Normalization Strategy: Bottom-up and Top-down design approaches.
Documentation: Detailed technical report in Greek, explaining every design decision
